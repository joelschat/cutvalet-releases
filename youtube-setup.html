Publishing to YouTube — one-time setup

Cut Valet uploads to your channel with your own (free) Google credentials. There is no shared "Cut Valet app" on Google's side — you create a tiny personal one in about ten minutes, and your videos never pass through anyone else's account or quota. You only do this once.

1 · Create a Google Cloud project

  1. Go to console.cloud.google.com and sign in with the Google account that owns your YouTube channel.
  2. Click the project picker (top bar) → New project. Name it anything — e.g. cut-valet-uploads — and create it.

2 · Enable the YouTube Data API v3

  1. With your new project selected, open APIs & Services → Library.
  2. Search for YouTube Data API v3 and click Enable.

3 · Set up the OAuth consent screen

  1. Open APIs & Services → OAuth consent screen.
  2. Choose External (the only option without a Google Workspace organisation) and fill in the two required fields: an app name (e.g. "Cut Valet") and your email address.
  3. You can skip scopes and branding — Cut Valet requests what it needs at sign-in time.
  4. Under Audience → Test users, add your own Google account. For personal use that's all you need.
The 7-day sign-in caveat. While the consent screen is in Testing mode, Google expires your saved sign-in after 7 days — Cut Valet will pop the browser consent window again each week. To make the sign-in permanent, go back to the OAuth consent screen and click Publish app (to "In production"). You do not need to complete Google's verification for a personal app: the first sign-in after publishing shows a "Google hasn't verified this app" warning — click Advanced → Go to (your app name) once, and you're done. After that, tokens no longer expire weekly.

4 · Create the Desktop-app OAuth client

  1. Open APIs & Services → Credentials → Create credentials → OAuth client ID.
  2. Application type: Desktop app. Name it anything.
  3. Click the download icon next to the new client and save the JSON file.

5 · Point Cut Valet at the file

  1. Save the downloaded JSON to ~/DVEA/config/youtube_credentials.json (on Windows: C:\Users\<you>\DVEA\config\youtube_credentials.json) — or keep it anywhere and set the path in the app: YouTube tool → Delivery settings → Credentials file.
  2. Load an export in the YouTube tool and click Upload to YouTube. Your browser opens Google's consent screen once; after you approve, Cut Valet caches the sign-in token and future uploads are hands-free.
Quota & privacy notes. Google gives every project 10,000 API "units" per day; one upload costs about 1,600 — roughly six uploads a day, far more than a daily vlog needs. Your credentials and token stay on your machine (~/DVEA/config/) and are never sent anywhere except to Google. Cut Valet requests only the YouTube upload/manage scopes it uses for publishing, captions, and thumbnails.
